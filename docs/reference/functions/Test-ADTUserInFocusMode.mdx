## SYNOPSIS

Tests whether the user is in focus mode.

## SYNTAX

```powershell
Test-ADTUserInFocusMode [<CommonParameters>]
```

## DESCRIPTION

This function tests whether the user is in focus mode, returning true/false, or null if the API is unavailable (older OS, etc).

## EXAMPLES

### EXAMPLE 1

```powershell
Test-ADTUserInFocusMode
```

Returns the logged on user is in focus mode or not, or null if the API is unavailable.

## PARAMETERS

### CommonParameters

This cmdlet supports the common parameters: -Debug, -ErrorAction, -ErrorVariable, -InformationAction, -InformationVariable, -OutVariable, -OutBuffer, -PipelineVariable, -Verbose, -WarningAction, and -WarningVariable. For more information, see [about_CommonParameters](http://go.microsoft.com/fwlink/?LinkID=113216).

## INPUTS

### None

### You cannot pipe objects to this function.
## OUTPUTS

### System.Boolean

### Returns true/false depending on whether the user is in focus mode or not.
## NOTES
An active ADT session is NOT required to use this function.
